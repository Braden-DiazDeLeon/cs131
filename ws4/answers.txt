extract ()

The extract() function is  for unpacking different archive formats. It is implemented with a for loop, a file test, and a case statement.

The line for archive in $* loops through every argument passed to the function so the user can run extract file1.zip file2.tar.gz and process multiple archives.
For each argument the if condition [ -f $archive ] checks if the argument is a regular file. If it is not a valid file, the function prints '$archive' is not a valid file!

When the file exists, case $archive in compares the filename against different extension patterns. 
Each pattern runs the matching extraction command. 
The *) default case prints that it does not know how to extract the archive if none of the extensions match. 
The esac ends the case statement, fi ends the file-checking if statement, and done ends the loop over the arguments.

up ()

The up() function moves up a requested number of parent directories.

The function starts by creating a variable d and setting limit=$1, where $1 is the first argument passed to the function. 
The for loop for ((i=1 ; i <= limit ; i++)) runs once for each directory level requested. On each iteration, d=$d/.. appends another parent-directory component to the path.

After the loop, d=$(echo $d | sed 's/^\///') removes the leading slash that was created when the first /.. was appended. 
The if condition [ -z "$d" ] checks if d is still empty, which can happen if the user did not provide a number. 
In that case, the function defaults to d=.., meaning go up one level.  cd $d changes into the computed parent directory.
